# planning/features/

One folder per new-capability plan, lowercase kebab-case: `planning/features/<slug>/`.

```
<slug>/
  README.md   # router: scope, read order
  PRD.md      # what/why
  STATUS.md   # current state, next step, blockers
  PLAN.md     # optional implementation sequence
```

Register every active folder in `planning/STATUS.md`. When complete, fold the outcome into `context/` and move the folder to `planning/archive/`. See `planning/.instructions.md` for the full rules.
