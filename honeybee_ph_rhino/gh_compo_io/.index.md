# gh_compo_io/ — component workers

One `GHCompo_<Name>` worker class per component (constructor takes `_IGH` + inputs, exposes `.run()`). `__init__.py` re-exports every worker so GHPython wrappers can `from honeybee_ph_rhino import gh_compo_io`.

## Domain subpackages

- `apertures/` — windows / frames / glazing
- `hvac/` — mechanical systems
- `shw/` — hot water & piping
- `shading/` — shading
- `program/` — loads / schedules
- `cert/` — PHI / Phius certification
- `openph/` — OpenPH

Loose `*.py` at this level are workers not belonging to a domain subpackage.

## Notes
- Each worker pairs with a GHPython wrapper in `../../honeybee_grasshopper_ph/src/HBPH - <Name>.py` and a registry entry in `../_component_info_.py`.
- IronPython-2.7-safe only; Rhino/GH calls via `../gh_io.py` (`IGH`). See `../../context/CODING_STANDARDS.md`.
