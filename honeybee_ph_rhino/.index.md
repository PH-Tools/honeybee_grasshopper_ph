# honeybee_ph_rhino/ — component logic (backend)

The **IronPython 2.7** backend: the worker classes that hold all component logic, plus the GH/Rhino seam and the registry. See `../context/ARCHITECTURE.md`.

## Contents

- `gh_compo_io/` — the component **workers** (`GHCompo_*`), grouped by domain subpackage. See `gh_compo_io/.index.md`.
- `gh_io.py` — the `IGH` interface; the single seam over all Rhino/Grasshopper API calls. Route everything through it.
- `_component_info_.py` — the registry: `RELEASE_VERSION`, `CATEGORY`, `SUB_CATEGORIES`, `COMPONENT_PARAMS`. A new/renamed component needs an entry here.
- `make_spaces/` — builds the PH space/volume/floor-segment geometry model from Honeybee rooms.
- `scripts/` — helpers.

## Notes
- Everything here must be IronPython-2.7-safe — see `../context/CODING_STANDARDS.md`.
- Tests for these workers live in the `honeybee_ph` repo, not here.
- `.vscode` fsdeploy auto-copies this folder into the Rhino site-packages and the PHX venv on save.
